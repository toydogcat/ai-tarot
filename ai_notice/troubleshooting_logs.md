# 🛠️ AI-Tarot Troubleshooting & Battle Lessons (除錯與戰鬥經驗紀錄)

本文件紀錄了 AI-Tarot 在開發與部署（特別是與 ai-factory 整合）過程中遇到的重大錯誤及其解決方案。

## 1. Docker Build DNS 與網路問題
- **錯誤現象**：在 `docker build` 階段，`pip install` 或 `npm install` 頻繁報錯 `Temporary failure in name resolution`。
- **原因**：部分 Docker 版本的 Bridge 網路與宿主機 DNS 配置（如 systemd-resolved）衝突。
- **解決方案**：
  - 在 `docker-compose.yml` 的 `build` 區塊加入 `network: host`。
  - 這能確保 Build 過程直接使用宿主機網路，解決 99% 的套件下載失敗問題。

## 2. Build Context 過大導致上傳緩慢
- **錯誤現象**：執行 `docker compose up --build` 時，顯示 `Sending build context to Docker daemon` 且大小高達數 GB。
- **原因**：沒有設定 `.dockerignore`，導致整個 `node_modules`、`history/` 和 `assets/` 都被打包進去。
- **解決方案**：
  - 建立 [.dockerignore](file:///home/toymsi/documents/projects/Github/ai-tarot/.dockerignore)。
  - 排除 `frontend/node_modules`、`backend/history`、`backend/assets` 等動態生成或靜態大檔案。
  - **成果**：打包大小從 3.1GB 降至 MB 級別。

## 3. Cloudflare Tunnel (PROD Test) 的坑
### A. 網址動態性
- **問題**：臨時隧道網址隨機，前端 API 路徑會失效。
- **解決方案**：
  - 實作 [catch_url.py](file:///home/toymsi/documents/projects/Github/ai-tarot/catch_url.py) 自動自動同步網址到 `.env.production`。
  - **進化版**：在 `main.js` 實作「環境感知」，只要網址列域名包含 `trycloudflare.com`，自動改用相對路徑 `/api`。

### B. 警告頁面攔截 (Interstitial Page)
- **問題**：第一次訪問時會跳出警告頁，導致 API 請求拿到 HTML 而報錯。
- **解決方案**：
  - 前端 Fetch 攔截器注入 `cf-skip-browser-warning: any` 標頭。

## 4. 房室網路隔離 (502 Bad Gateway)
- **錯誤現象**：隧道啟動後訪問報 502。
- **原因**：隧道 Container 與 房間 Container 沒在同一個 Docker Network 中，或房間沒設定 `restart: unless-stopped`。
- **解決方案**：確保所有連動服務都在 `tarot-net` 中，並為後端加入 `healthcheck` 以確保隧道在後端完全就緒後才啟動。
