---
description: 自動化 TDD 迴圈：執行指定指令，失敗則立刻自動修復直至成功。
---

這個 Workflow 是為了讓大獅子 (AI) 發揮完全潛力而設計。它可以被用來跑單元測試、E2E 測試，或是執行任何可能會吐出 Error 的腳本。遇到錯誤時，AI 會自己除錯並修改程式碼，直到亮綠燈通過。

// turbo-all
1. 執行使用者指定的目標指令 (例如 `pytest tests/test_admin.py` 或 `python main.py`)。若使用者起手式沒有指明測試目標，請先向他確認要執行哪段程式碼。
2. 觀察終端機指令的輸出內容與退出狀態碼。
3. 如果執行完全成功 (無 ERROR 或 FAILED 訊息)，向使用者回報「全部測試完美通過」，並結束此工作流。
4. 如果遇到 Error 或 FAILED，仔細閱讀終端機印出的 Traceback 與錯誤追蹤訊息。
5. 思考錯誤發生的原因，提出具體的修復假說。
6. 使用強大的 `replace_file_content` 或是編輯工具，直接修改有 Bug 的程式碼檔案。
7. 【非常重要】修復完成後，自動回到這個 Workflow 的「步驟 1」重新執行相同的測試指令。
8. 不斷循環此「執行 ➔ 除錯 ➔ 修改 ➔ 執行」過程，直到測試完全通過為止。(若遇上同一問題反覆修改 3 次仍失敗的絕境，請停止動作並呼叫小獅子/人類工程師請求指導)。
